# 项目配置

## dotenv 环境变量配置

在 FastAPI 中，获取与设置环境变量主要有 2 种方式：

- **`os` 模块 + CMD 命令行设置：直接自动识别输出**
- **`.env` 文件 + `python-dotenv` 依赖：自定义配置**

在实际项目中，把环境变量写在终端很不方便。我们通常会在项目根目录下创建一个 `.env` 文件：

```ini
# .env 文件内容
ENV=production
DATABASE_URL=localhost:5432
```

然后利用第三方库 `python-dotenv` 自动加载它：

1. 先安装：`pip install python-dotenv`
2. 在代码中引入：

```Python
import os
from dotenv import load_file, load_dotenv

# 加载 .env 文件中的变量到系统中
load_dotenv()

# 现在可以正常获取了
env = os.getenv("ENV", "development")
print(env)  # 输出: production
```

## static 静态文件配置

### 为什么需要它？

虽然在生产环境中，纯粹的前后端分离项目通常会把前端静态资源交给 **Nginx**、**CDN** 或者 **Vercel/Netlify** 等专门的服务，但是有些情况下（如中小型项目初期的文件托管），则需要存储到后端服务器中。

| **阶段/环境**        | **静态资源处理方式** | **核心原因**                                                 |
| -------------------- | -------------------- | ------------------------------------------------------------ |
| **开发环境 (Local)** | **FastAPI 静态配置** | 简单快捷，无需在本地配置复杂的 Nginx，避免跨域麻烦。         |
| **中小型私有化部署** | **FastAPI 静态配置** | 减少运维成本，客户开箱即用，降低部署容器的数量。             |
| **高并发生产环境**   | **Nginx / CDN 转发** | 性能考虑。Python (ASGI) 处理高并发静态文件的效率远不如 Nginx (C+异步内核) 或 CDN 边缘节点。 |

还有一种情况：

FastAPI 默认的 `/docs` (Swagger UI) 和 `/redoc` 极其好用，但它们**默认是从 CDN 加载前端 JS/CSS 脚本**的。

- **痛点**：在**内网开发、断网环境**、或者**企业私有化部署**时，经常会**出现 `/docs` 打开后一片空白**（因为连不上外网 CDN）
- **解法**：需要把 **Swagger UI 的静态文件下载到本地**，然后通过 **FastAPI 的静态文件配置进行本地托管**，从而彻底**脱离外网限制**

### 开始使用

在 FastAPI 中配置静态文件（如 CSS、JavaScript、图片等）非常简单。FastAPI 提了一个专门的模块 **`StaticFiles`** 来处理。

#### 核心概念

首先，需要提取在**项目根目录**下，**创建一个存放静态文件的文件夹，默认命名为 `/static`**。

```python
fastapi_app/
|----- static/ # 存储静态文件（CSS、图片、音视频...）
|----- routes/ # 存放 APIRouter 接口路由的模块
|----- server.py # 主入口程序文件
```

#### StaticFiles 类

`StaticFiles` 是 由 `fastapi.staticfiles` 模块提供的一个类，专门用于将指定目录挂载到当前 FastAPI 应用身上，构建 URL 静态资源路径的映射关系。

```python
from fastapi.staticfiles import StaticFiles

StaticFiles(directory=<需挂载的静态文件目录路径>)
```

#### app.mount() 挂载

核心定义：通过 FastAPI 实例的 **`mount()` 方法，传入 `StaticFiles()` 类**来构建 **静态文件目录** 与 **静态资源 URL 路径** 的**映射关系**。

```python
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles

app = FastAPI()

# 挂载静态文件目录
app.mount(path=<静态资源的 URL 路由路径>, app=StaticFiles(directory=<需挂载的静态文件目录路径>), name=<别名>)
```

参数：

- **`path`**：**静态资源的 URL 路由路径**（URL 前缀）。用户在浏览器中访问静态文件的 URL 会以这个开头

  ​	例如：`http://127.0.0.1:8000/static/style.css`。

- **`app`**：**静态资源的本地目录**。通常为 `/static`，也可以是别的目录

- **`name`**： FastAPI 内部使用的**别名**，主要用于在模板（如 Jinja2）中通过 `url_for` 动态生成静态文件的 URL

#### 示例

访问资源：

- http://127.0.0.1:8001/static/AI%20%E6%96%B9%E6%A1%88.md
- http://127.0.0.1:8001/static/1.png
- ...

项目结构：

```python
fastapi_app/
|----- static/ # 存储静态文件（CSS、图片、音视频...）
|----- routes/ # 存放 APIRouter 接口路由的模块
|----- server.py # 主入口程序文件
```

`server.py`：

```python
# 主入口文件
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
# 导入路由
from router import users_api, product_api


# 主应用
app = FastAPI()

# 挂载静态资源目录
app.mount("/static", StaticFiles(directory="static"), name="static")

# 导入路由，并添加到主应用中，路由前缀为 /users，Swagger UI 中显示的标签为 用户管理
app.include_router(users_api.router, prefix='/users',
                   tags=['用户管理'], responses={404: {"description": "Not Found"}})

# 导入路由，并添加到主应用中，路由前缀为 /product，Swagger UI 中显示的标签为 商品管理
app.include_router(product_api.router, prefix='/products',
                   tags=['商品管理'], responses={404: {"description": "Not Found"}})


# 启动服务
if __name__ == '__main__':
    import uvicorn

    uvicorn.run(app='server:app', host='127.0.0.1', port=8001, reload=True)
```

## FastAPI() 应用配置

在 FastAPI 中，`FastAPI()` 实例不仅是整个应用的入口，还可以通过传入不同的配置参数来控制应用的**元数据、OpenAPI 文档生成、中间件、异常处理**等。

常见配置项：

| **参数名**        | **类型**     | **默认值**        | **作用说明**                                          |
| ----------------- | ------------ | ----------------- | ----------------------------------------------------- |
| **`title`**       | `str`        | `"FastAPI"`       | API 的名称                                            |
| **`description`** | `str`        | `""`              | API 的描述（支持 Markdown）                           |
| **`version`**     | `str`        | `"0.1.0"`         | API 的版本号                                          |
| **`docs_url`**    | `str | None` | `"/docs"`         | Swagger UI 路径，`None` 为关闭                        |
| **`redoc_url`**   | `str | None` | `"/redoc"`        | ReDoc 路径，`None` 为关闭                             |
| **`openapi_url`** | `str | None` | `"/openapi.json"` | OpenAPI JSON 路径                                     |
| **`root_path`**   | `str`        | `""`              | 代理背后的根路径前缀                                  |
| **`lifespan`**    | `Lifespan`   | `None`            | 应用生命周期管理器（启动/关闭事件）                   |
| **`debug`**       | `bool`       | `False`           | 是否开启调试模式（通常用 uvicorn 的 `--reload` 代替） |

以下是 FastAPI 常见配置项的分类详解：

### 基础元数据配置（MetaData）

这些配置主要用于定义 **整个 API 微服务的基本信息**，会**直接展示在 `/docs` (Swagger UI) 或 `/redoc` 页面上**。

```python
from fastapi import FastAPI

app = FastAPI(
    title="我的电商系统 API",
    description="这是一个用于管理商品和订单的后台 API 系统。",
    version="1.0.0",
    terms_of_service="https://example.com/terms/",
    contact={
        "name": "技术支持团队",
        "url": "https://example.com/support",
        "email": "support@example.com",
    },
    license_info={
        "name": "Apache 2.0",
        "url": "https://www.apache.org/licenses/LICENSE-2.0.html",
    },
)
```

#### 参数说明

- **`title`**: API 的标题（字符串）
- **`description`**: API 的详细描述，支持 **Markdown** 语法
- **`version`**:  API 版本号（如 `1.0.0`）
- **`terms_of_service`**: 服务条款的 URL 链接
- **`contact`**: 联系人信息的字典（包含 `name`, `url`, `email`）
- **`license_info`**: 许可证信息的字典（包含 `name`, `url`）

### 自动化文档配置 (Docs & OpenAPI)

如果需要**关闭自动文档**，或者**修改文档的访问路径**，可以使用这些参数。

- **`docs_url`**：Swagger UI 的**访问路径，默认为 `"/docs"`**。如果**设置为 `None`，则禁用 Swagger UI**

- **`redoc_url`**：ReDoc 的**访问路径，默认为 `"/redoc"`**。如果**设置为 `None`，则禁用 ReDoc**

- **`openapi_url`**：**OpenAPI 规范 JSON 文件的路径，默认为 `"/openapi.json"`**。

  ​	设置为 `None` 则关闭整个 OpenAPI 生成（同时也会导致 docs 无法使用）

- **`openapi_tags`**：用于**为不同的路由标签（Tags）添加元数据和描述**，效果等价于 `@app.post(tags=,description=)`

示例：

```python
# 示例：在生产环境中禁用文档
import os
from fastapi import FastAPI

app = FastAPI(
    docs_url=None if os.getenv("ENV") == "production" else "/docs",
    redoc_url=None if os.getenv("ENV") == "production" else "/redoc",
    openapi_tags=[
        {
            "name": "users",
            "description": "用户管理相关接口，包括注册、登录、信息修改。",
        },
        {
            "name": "items",
            "description": "商品管理相关接口。",
        }
    ]
)
```

### 路由与服务器配置

控制 API 整体的路由行为和基础 URL。

- **`root_path`**：**Nginx 反向代理的应用根路径前缀。即 Nginx 配置文件中 location /api/v1 {} 中的 /api/v1 会指向当前应用**

  ​	如果 **FastAPI 应用部署在反向代理（如 Nginx、Traefik）后面**，并且有**前缀**（例如 `https://example.com/api/v1/`），需要设置 `root_path="/api/v1"`，否则自动文档会无法正确发起请求。

- **`root_path_in_servers`**：默认为 `True`。是否将 `root_path` 自动包含在 OpenAPI 的 `servers` 列表中。

- **`redirect_slashes`**：默认为 `True`。是否自动处理末尾的斜杠。

  ​	例如访问 `/items/` 时，若路由只定义了 `/items`，会自动重定向

```python
app = FastAPI(root_path="/api/v1")
```

### 生命周期与事件 (Lifespan)

用于在 FastAPI **应用启动（Startup）和关闭（Shutdown）**时**执行特定逻辑（钩子函数）**（如连接/断开数据库、初始化缓存等）。

> ⚠️ **注意**：原有的 `on_event("startup")` 和 `on_event("shutdown")` 已不推荐使用，**推荐使用最新的 `lifespan` 参数**。

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI

@asynccontextmanager
async def lifespan(app: FastAPI):
    # 【启动时执行】
    print("服务启动中...")
    yield
    # 【关闭时执行】
    print("服务已关闭...")

app = FastAPI(lifespan=lifespan)
```
